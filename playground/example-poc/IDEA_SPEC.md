## 🎯 概要（エレベーターピッチ）

**これは新しいPOCを始める時のテンプレートです。このファイルをコピーして、あなたのアイデアに合わせて編集してください。**

## 使い方

### 1. このテンプレートをコピー
```bash
# 新しいPOCディレクトリを作成
mkdir playground/your-project-name

# テンプレートをコピー
cp playground/IDEA_SPEC_TEMPLATE.md playground/your-project-name/IDEA_SPEC.md
cp playground/example-poc/README.md playground/your-project-name/README.md
```

### 2. IDEA_SPEC.mdを編集
- プロジェクト名、ステータス、日付を更新
- 概要セクションを記入（1-2文）
- 問題・課題を明確化
- 解決策を記述
- 主要機能をリストアップ

### 3. 開発開始
```bash
cd playground/your-project-name

# 必要なディレクトリを作成
mkdir -p src tests docs

# 開発開始！
```

### 4. 定期的に更新
- 実装が進んだら「実装状況」セクションを更新
- 課題が見つかったら「課題・リスク」に追加
- 新しいアイデアは「メモ・アイデア」に記録

## ヒント

### アイデアの書き方
✅ **Good**: 「プレイヤーとの会話から学習し、より良いストーリーを生成するゲーム」
❌ **Bad**: 「AIを使ったゲーム」

### 問題の定義
✅ **Good**: 「従来のAIゲームは固定的な応答パターンで、ユーザーフィードバックを活かせない」
❌ **Bad**: 「ゲームが面白くない」

### 機能の記述
✅ **Good**:
```
1. データ収集パイプライン
   - プレイヤーとの会話を自動記録
   - 高評価のインタラクションを抽出
```
❌ **Bad**:
```
1. データを集める
```
