# Tokyo Risk - Multi-Agent System Quick Start

## 🎮 すぐに始める

### 1. サーバー起動
```bash
cd /Users/soki/Gemini-3-Tokyo-Hackathon/playground/tokyo-risk
uvicorn server:app --port 8766 --reload
```

### 2. ブラウザでアクセス
```
http://localhost:8766
```

### 3. プレイ手順
1. 「ゲーム開始」ボタンをクリック
2. プロンプト設定ダイアログで戦略を設定
   - **全体戦略**: 全兵士共通の指示
   - **個別指示**: 各兵士専用の追加指示（オプション）
3. 「保存してゲーム開始」をクリック
4. 地図上の青い矢印（プレイヤー）と赤い矢印（AI）を観察

---

## 🤖 エージェントシステムの仕組み

### 各陣営に10体のAIエージェント
- **プレイヤー側**: player_001 ~ player_010 (青い矢印)
- **AI側**: ai_001 ~ ai_010 (赤い矢印)

### 自律的な行動判断
- **Gemini API使用**: 001-003番のエージェントが自律的に判断
- **ルールベース**: 004-010番のエージェントがシンプルな行動パターン
- **判断頻度**: 5秒ごと（コスト最適化のため）

### リアルタイム移動
- **位置更新**: 200ms間隔で自動更新
- **スムーズ移動**: 目的地に向かって連続移動
- **方向表示**: 矢印が進行方向を向く

## 📝 プロンプト設定のコツ

### 効果的なプロンプト例

#### 全体戦略プロンプト
```
あなたは東京23区の陣取りゲームの兵士です。
目標: 敵の区を占領し、味方の支配領域を拡大する。

【行動原則】
1. 味方と協力して行動する
2. 孤立した敵を優先的に攻撃
3. 防御が薄い区を積極的に占領
4. 体力が少ない時は味方の近くで回復
```

#### 個別指示の例
- **兵士001**: `偵察専門。敵の動きを観察し、危険を報告する`
- **兵士002-003**: `攻撃特化。積極的に敵陣に進軍する`
- **兵士004-005**: `防御重視。味方の領地を守り、敵の侵入を阻止する`
- **兵士006-007**: `機動力重視。素早く移動して弱点を突く`

### プロンプトの差別化が勝敗を分ける
- より具体的な指示 → より戦略的な行動
- 役割分担 → 効率的な連携
- 状況判断の基準 → 柔軟な対応

## 📊 コスト管理

### 現在の設定
- API使用: 6体（3体×2陣営）
- 判断頻度: 5秒ごと
- 1ゲーム30分 ≈ 360回のAPI呼び出し
- 推定コスト: **約$0.004/ゲーム**

### コスト削減のヒント
1. API使用エージェント数を減らす（3体→2体）
2. 判断頻度を下げる（5秒→10秒）
3. ゲーム時間を短くする
4. ルールベースの精度を上げる
